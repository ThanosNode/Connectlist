Application Overview
ConnectList is a Web3-focused web platform inspired by Craigslist, enabling personal connections, job opportunities, and business services. It features a minimalist UI with a white background and blue links, prioritizing simplicity, security, and cryptocurrency payments for a modern user experience tailored to Web3 enthusiasts.
Core Architecture
Technology Stack
Frontend: React (18.x) with TypeScript, TailwindCSS (3.x), shadcn UI components for consistent design
Backend: Node.js (20.x) with Express.js (4.x) for RESTful API
Database: PostgreSQL (16.x) with Drizzle ORM for schema management and migrations
Authentication: Session-based with JWT support, bcrypt (v5.x) for password hashing
Payments: Coinbase Commerce API for cryptocurrency payments (BTC, ETH, USDC)
Caching: Redis (7.x) for high-performance data retrieval
Testing: Jest for unit tests, Cypress for E2E tests
Key Components
Client-Server Architecture: React SPA communicates with Express API via RESTful endpoints
Database Schema: Relational model with foreign keys and indexes for performance
Security Layer: Middleware for authentication, rate limiting, and permission checks
Payment System: Coinbase Commerce for secure crypto transactions
Caching Layer: Redis for caching listings and user sessions
Feature Set
User Authentication System
Registration: Email-based with security question verification, rate-limited to 5 attempts per hour
Login/Logout: JWT-based authentication with refresh tokens, session stored in Redis
Security: Bcrypt-hashed passwords, secure cookies (HttpOnly, SameSite=Strict)
Account Types: Regular users, sub-admins (limited permissions), super admins (full control)
Content Management
Personal Listings
Free to post, 30-day expiration
Optional $5 featured listing (red font marker, prioritized display)
Location-based (country/state/city), with email verification for spam protection
Paginated results (20 listings per page)
Job Listings
$15 posting fee, 30-day expiration
Country-specific categorization, displayed at top of category pages
Supports remote/in-person filtering
Business Listings
Free for first 30 days, then $5/month subscription
Tiered plans (1, 3, 6 months) with auto-renewal and cancellation options
Location-based categorization, with premium badge for paid subscribers
Administrative Functions
User Management: Super admins create/delete sub-admins, view user activity
Content Moderation: Flagging system, post deletion, automated spam detection
Financial Dashboard: Tracks revenue, transaction history, and category breakdowns
Security: Logs all admin actions, enforces strict permissions
Payment Processing
Coinbase Commerce: Supports BTC, ETH, USDC with webhook for payment status (pending, confirmed, failed)
Pricing: $15 for job postings, $5 for featured listings, $5/month for business subscriptions
Subscription Management: Handles renewals, cancellations, and failed payment retries
Financial Reporting: Admin dashboard with exportable CSV reports
Security Implementation
Authentication Security
Password Handling: Bcrypt with 12 rounds, automatic upgrades for legacy hashes
Session Management: JWT with 1-hour expiry, refresh tokens stored in Redis
Session Expiration: Rolling sessions extend with activity, max 7 days
Logout: Clears JWT and Redis session, invalidates refresh tokens
Data Protection
Input Validation: Joi for server-side validation, React Hook Form for client-side
XSS Prevention: React's built-in escaping, DOMPurify for user inputs
CSRF Protection: CSRF tokens in all POST/PUT/DELETE requests
Database Security: Row-level security, parameterized queries, no raw SQL
Admin Access Control
Permission Enforcement: Database-level checks for admin privileges
Tiered Permissions: Super admins manage all, sub-admins moderate content
Access Logging: Logs admin actions with timestamps and IP addresses
Failed Attempts: Rate-limited to 10 attempts per hour, logged for auditing
Internationalization Support
Multi-Country Support: 18 countries with region-specific settings (e.g., US states, EU regions)
Localization: Multi-language UI (English, Spanish, French) via i18next
Formatting: Region-appropriate dates, times, and currency (dynamic conversion via Coinbase API)
Search: Location-based filtering with geolocation support
Technical Implementation Details
Database Schema
Users: id (PK), email, password_hash, security_question_hash, security_answer_hash, is_admin, created_at
Listings: id (PK), user_id (FK), title, description, location, is_featured, expires_at
Jobs: id (PK), user_id (FK), title, description, location, salary, is_remote, expires_at
Payments: id (PK), user_id (FK), listing_id (FK), amount, currency, status, coinbase_charge_id
Sessions: id (PK), user_id (FK), jwt_token, refresh_token, expires_at
Reports: id (PK), listing_id (FK), user_id (FK), reason, status
Indexes: On user_id, location, expires_at for query performance
API Endpoints
GET /api/listings: Fetch paginated listings, supports filters (location, category)
POST /api/listings: Create a listing, requires JWT
PUT /api/listings/:id: Update listing, requires ownership
DELETE /api/listings/:id: Delete listing, requires ownership or admin
POST /api/auth/register: Register user with email and security question
POST /api/auth/login: Login with JWT response
POST /api/payments/create-charge: Initiate Coinbase payment
GET /api/admin/financials: Admin-only revenue report (requires super admin)
Rate Limiting: 100 requests/hour for public endpoints, 10/hour for sensitive (e.g., login)
Security Measures
Environment Variables: Managed via .env with dotenv, secrets in AWS Secrets Manager
Error Handling: Standardized { error: string, code: number } response format
Session Security: Redis-backed sessions, encrypted JWT payloads
Input Sanitization: Parameterized queries, validated inputs with Joi
Recent Security Issues and Fixes
Improper Admin Access:
Problem: Development auto-login granted admin access to regular users
Fix: Removed auto-login, added middleware to verify is_admin from database, created admin@example.com with proper permissions
Session Management:
Problem: Sessions not persisting between requests
Fix: Implemented Redis session store, added rolling: true, standardized cookie settings
Security Questions:
Change: Replaced reCAPTCHA with security questions
Implementation: Hash answers with bcrypt, rate-limit verification attempts
